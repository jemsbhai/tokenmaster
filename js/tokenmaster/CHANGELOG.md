# Changelog

All notable changes to the tokenmaster npm package are documented in this
file.

## 0.1.0 (2026-07-08)

- Complete TypeScript port of the Python 0.1.0 reference, with full API
  parity.
- Typed data model (ModelProfile, TurnUsage, MeterState, CalibrationRecord)
  with provenance tags, validation, and exact JSON round trips.
- Meter: latest-turn accounting, EWMA velocity with incremental variance,
  conservative turns-to-exhaustion, zone logic, exhaustion semantics, and
  persistence by replay.
- Bundled model registry (12 models, dated and cited pricing), alias and
  dated-suffix resolution, user overrides, Meter.forModel. The snapshot is
  embedded at build time from the canonical models.json, so there is no
  filesystem access at runtime.
- Typed event stream with deterministic per-turn emission: TurnRecorded,
  ZoneChanged, VelocityShift, ModelChanged, AdvisorRecommendation,
  HandoffEvaluated; wire round trips via eventFromDict.
- Advisor: ThresholdPolicy baseline, PredictivePolicy (conservative-eta
  coverage with fallback delegation), CostModelPolicy (cache break-even k*,
  dual-unit ledger, feasibility handling).
- Handoff fidelity protocol: probe categories, weighted scoring, adapter
  interfaces for every LLM touchpoint, ExactMatchJudge, reproducibility
  fields and explicit caveats.
- Conformance: replays all nine spec/ vectors generated by the Python
  reference; states, events, provenance strings, and doubles match.
- Packaging: dual ESM and CommonJS builds with bundled TypeScript
  declarations; zero runtime dependencies.
